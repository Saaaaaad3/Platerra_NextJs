# Development Rules

## Philosophy

This project is an MVP.

Always prioritize:

1. Simplicity
2. Speed of delivery
3. Maintainability

Avoid:

- Overengineering
- Premature optimization
- Microservices
- Complex abstractions

## Current Stack

- Next.js
- TypeScript
- Local database (temporary)
- PostgreSQL (future)
- AI integration (future)

## Architecture Rules

- Prefer simple solutions.
- Keep frontend and backend in the same project.
- Avoid introducing new dependencies unless justified.
- Optimize for shipping quickly.

## UI Rules

- Mobile-first.
- Fast loading.
- Food photography is a first-class feature.
- Clean and modern design.
